# LiberSystem service and artifact manifest - the single source of truth for the boot
# chain. ServiceManager's dependency table (each managed service's name and the services
# that must be up before it may start) is generated from the `service` and `instance`
# rows below; the kernel build script derives its init.pkg and system-volume staging
# lists from every row. Adding a normal service is one row here plus the service's own
# code - the runtime table and the staging lists can no longer drift apart.
#
# columns: kind  name  crate  stage  [policy/providers...]
#   kind : launcher | service | instance | probe | driver | tool | dynamic | component
#          - service : a managed service with its own ELF (ServiceManager supervises it)
#          - instance: a managed service backed by another program's ELF (no own binary)
#          - launcher: spawned from the init package, not supervised as a managed service
#          - probe   : a self-test helper spawned by the supervisor
#          - driver / tool / component: statically linked staged programs
#          - dynamic: staged ET_DYN programs built by the system-image linker
#   crate: crate directory under src/user (or '-' for an instance with no own ELF)
#   stage: pinned (kept in init.pkg) | volume (staged on the system volume) | none
#   restart: the supervisor's crash policy (service and instance rows only) -
#          transparent (restart per the ladder; clients re-resolve through the broker)
#          or escalate (record Failed; today's default until a service migrates)
#   deps : the services that must be Running first (service and instance rows only)
#   providers: direct prefix-free library names for dynamic rows

# managed services - ServiceManager brings these up in dependency order. The row order

# LogService, though all depend on it): the dependency resolver derives the real order,
