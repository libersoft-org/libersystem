# LiberSystem service and artifact manifest - the single source of truth for the boot
# chain. ServiceManager's dependency table (each managed service's name and the services
# that must be up before it may start) is generated from managed-service rows
# rows below; the kernel build script derives its init.pkg and system-volume staging
# lists from every row. Adding a normal service is one row here plus the service's own
# code - the runtime table and the staging lists can no longer drift apart.
#
# source rows: source  logical-owner  workspace-relative-crate-path
# artifact columns: kind  name  crate  stage  [policy/features/providers...]
#   kind : launcher | service | dynamic-service | instance | probe | driver | tool | dynamic
#          - service : a managed service with its own ELF (ServiceManager supervises it)
#          - dynamic-service: a managed ET_DYN service; `--` separates deps/providers
#          - instance: a managed service backed by another program's ELF (no own binary)
#          - launcher: spawned from the init package, not supervised as a managed service
#          - probe   : a self-test helper spawned by the supervisor
#          - driver / tool: statically linked staged programs pending migration
#          - dynamic: staged ET_DYN programs built by the system-image linker
#   crate: logical source owner declared by one source row (or '-' for an instance)
#   stage: pinned (kept in init.pkg) | volume (staged on the system volume) | none
#   restart: the supervisor's crash policy (managed service and instance rows only) -
#          transparent (restart per the ladder; clients re-resolve through the broker)
#          or escalate (record Failed; today's default until a service migrates)
#   deps : the services that must be Running first (managed service and instance rows)
#   features: `-` for defaults or a comma-separated Cargo feature set for library rows
#   providers: direct prefix-free library names for library and dynamic rows

# Physical Cargo roots are explicit and independent of logical artifact identities.
# Paths are relative to src/ and change during physical layout migrations; artifact rows
# continue to name logical owners so path components never become runtime identities.
